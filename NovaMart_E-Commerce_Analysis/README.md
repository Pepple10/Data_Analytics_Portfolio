**NovaMart Sales Analysis Dashboard**



**Project Overview**



In this project, I analyzed sales data from an e-commerce company called NovaMart. My goal was to take messy raw business data and turn it into clear insights that a company could use to make better decisions.



When I first looked at the dataset, I immediately noticed several issues. The data was spread across multiple tables, some values were inconsistent, and several columns contained errors or missing information. Before any analysis could happen, I needed to clean and structure the data properly.



To solve this, I built a workflow using Excel, MySQL, and Power BI to clean the data, organize it into a relational structure, and finally create a dashboard that highlights key business insights.



**Business Problem**



While exploring the dataset, I asked a few key questions a business would typically want answered:



* How much revenue is the company generating?
* Which product categories perform the best?
* How do customers prefer to pay?
* Are there regions that generate more sales than others?
* Are there any patterns in sales trends over time?



Without a structured analysis, these answers are hidden inside raw data.



My goal was to uncover those insights and present them in a clear visual dashboard





**Tools I Used**



To complete this project, I used three main tools:



**Excel**

* Cleaned inconsistent values
* Corrected formatting errors
* Standardized columns such as state codes and payment fields



**MySQL**

* Imported the dataset into a structured database
* Organized the data across multiple related tables
* Ran queries to explore the data and verify relationships



**Power** **BI**

* Connected to the SQL database
* Built relationships between tables
* Created an interactive dashboard to visualize insights.





**Data Cleaning Process**



When I started exploring the dataset, I noticed several problems:



Some state values were inconsistent or incorrect

* Certain columns had missing values
* Some records had formatting issues
* The dataset contained duplicate or messy entries



To fix this, I used Excel to:

* Standardize inconsistent values
* Replace incorrect entries
* Clean text formatting
* Ensure columns had the correct data types



After cleaning the data, I imported it into MySQL, where I structured it into relational tables and confirmed the relationships between customers, orders, payments, and products



**Dashboard Metrics**



To give a quick overview of business performance, I created several key performance indicators:



* Total Revenue
* Total Orders
* Total Customers
* Average Order Value



These metrics help summarize the company’s overall performance.





**Insights I Explored**



Through the dashboard, I explored several areas of the business:



* Sales performance over time
* Which product categories generate the most revenue
* Customer payment preferences
* Geographic distribution of customers and sales



The dashboard allows these insights to be explored interactively.





**Business Recommendations**



Based on the analysis, a business like NovaMart could:



* Focus marketing on the best-performing product categories
* Optimize checkout systems around the most used payment methods
* Expand operations in regions with strong sales demand
* Investigate periods where sales drop to identify potential issues



**What I Learned From This Project**



This project helped me practice the full workflow of a data analyst:



* Cleaning messy datasets
* Structuring relational databases
* Building data models
* Creating interactive dashboards
* Translating data into business insights



It also reinforced how important data preparation is before meaningful analysis can happen.
